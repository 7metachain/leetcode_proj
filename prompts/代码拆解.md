# 🧠 通用刷题 Prompt（关卡化 · 单骨架 · 本地可运行）

请不要直接给我最终解法。
请把这道算法题设计成一个**"单文件、逐步填空、关卡化推进"的学习骨架**，让我通过一步步补全代码，最终自然写出最优解。

---

## 🎯 总目标（必须达成）

- 所有关卡 **共用一个代码骨架文件**
- 每一关都能在本地直接运行
- 我在每一关只需在指定 `TODO` 位置填写极少量代码
- 当我完成最后一关时，这个骨架文件 **自然变成最终最优解**
- 过程中我能清楚理解：
  - 题目建模
  - 正确性来源
  - 优化动机
  - 关键思想（贪心 / DP / 指针等）

---

## 🧱 骨架设计强制要求（非常重要）

### 1️⃣ 单文件骨架（Single Skeleton）

- 只生成 **一个完整代码文件**
- 文件中通过 `LEVEL`（或类似开关）控制当前运行到第几关
- 不允许"每一关一份新代码"

### 2️⃣ 每一关都必须本地可运行

每个 Level 下：
- 程序可以直接运行 / 编译
- 内置至少 1 个固定测试样例
- 能输出结果用于验证
- **禁止伪代码、禁止省略实现**

### 3️⃣ 逐关填空（Incremental TODO）

- 每一关只新增或替换 1–3 行核心逻辑
- 用清晰的 `TODO (Level X)` 标注我需要填写的位置
- 禁止推倒重写已有代码
- 旧代码必须被反复复用（刻意重复）

### 4️⃣ 关卡顺序必须是（不可打乱）

1. 题意建模（公式 / 函数表达）
2. 暴力或朴素正确解（作为真值）
3. 中间信息暴露（打印 / explain / 日志）
4. 解法结构骨架（如双指针 / DP / BFS 框架）
5. 关键思想注入（贪心选择 / 状态转移）
6. 最优解完成
7. （可选）随机对拍 / 小优化 / 模板封装

### 5️⃣ 重复优先于简洁

- 关键变量名、关键计算公式、关键结构 **跨关卡重复出现**
- 禁止为了"好看"而提前抽象或省略
- 目标是形成 **代码级肌肉记忆**

### 6️⃣ 统一变量体系

- 所有关卡使用同一套变量名
- 中途不得改名（除非明确教学目的）
- 降低理解与切换成本

---

## 🧩 骨架文件内部推荐结构（抽象）

1. **LEVEL 控制器**（决定跑到哪一关）
2. **核心基础函数**（如 `area` / `transition` / `check`）
3. **暴力解**（作为真值或对拍）
4. **目标解函数**（逐步从空 → 最优）
5. **统一测试入口**（`main` / `if __name__ == "__main__"`）
6. **明确标注的 `TODO (Level X)` 填空区**

---

## 🌍 语言要求

- 默认 **Python + C++ 双语并行**
- 两个骨架文件结构、Level、样例、输出语义必须一致
- 均需能在本地直接运行 / 编译

---

## 🧑‍🏫 交互规则

- 一次只生成一个骨架文件
- 不要一次性给出所有关卡的答案
- 允许我在每一关自己填写代码
- 我可以随时贴出当前骨架代码让你检查
- 你最多只指出 1–2 个关键问题，不直接代写

---

## ✅ 最终验收标准

当我完成所有 `TODO` 后：
- 文件中不再需要 `LEVEL` 限制
- 代码即为该题的最优解
- 我能不看答案，独立重新写出解法

---

## 🧠 一句话总结这个 Prompt 的精神

> **不要教我"答案"，而是给我一副能一步步拼成答案的骨架。**
